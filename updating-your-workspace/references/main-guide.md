# Main Agent (小丹) 指南

main workspace 的内容定位规则。

---

## 核心理解

**这个 skill 的作用**：从 memory/ 日志中提炼有价值信息，更新到正确的文件。

**更新来源**：memory/ 日记、research、insights、LEARNING、ERROR、FEATURE_REQUEST

**更新目标**：MEMORY.md、AGENTS.md、skills/

**更新频率低**：USER.md（静态人设，除非 Dan 明确说改）

---

## 文件定位

| 文件 | 定位 | 更新方式 |
|------|------|----------|
| **USER.md** | 人设（静态） | ❌ 不用这个skill更新 |
| **MEMORY.md** | 小丹业务的思维模式、个人IP打造的战略思维（记忆的北极星、工作方向、中长期目标） | ✅ 更新目标 |
| **AGENTS.md** | 重要工作规则 | ✅ 更新目标 |
| **skills/** | 知识库（特定知识，按需触发） | ✅ 更新目标 |
| **memory/research/** | 调研笔记 | 📥 更新来源 |
| **memory/insights/** | 深度洞察 | 📥 更新来源 |
| **memory/*.md** | 每日日记 | 📥 更新来源 |

---

## 内容分流决策树

```
从 memory/ 发现新知识
    ↓
1️⃣ 是 Dan 的人设改变？
    ├─ 是 → 告诉 Dan，等他确认后改 USER.md
    └─ 否 ↓

2️⃣ 是业务战略思维/工作方向/中长期目标？
    ├─ 是 → MEMORY.md
    │      例："业主视角是核心策略"、"内容迭代优于完美"
    └─ 否 ↓

3️⃣ 是重要工作规则/流程？
    ├─ 是 → AGENTS.md
    │      例："每次 heartbeat 先 git 备份"
    └─ 否 ↓

4️⃣ 是可复用的特定知识？
    ├─ 是 → skills/content-creator-knowledge/references/
    │      ├─ 小红书爆款 → xiaohongshu-guide.md
    │      ├─ 公众号结构 → wechat-guide.md
    │      └─ 发布策略 → publishing-strategy.md
    └─ 否 → 留在 memory/，不升级
```

---

## 具体例子

### 写入 MEMORY.md（战略思维）

```markdown
## 战术执行

### 内容策略
| 维度 | 方向 |
|------|------|
| 视角 | 从"业主"角度出发（吸引客户） |
| 形式 | 先图文，后续做视频 |
| 平台 | 公众号 > 小红书 > X |
```

**特征**：北极星、工作方向、中长期目标

### 写入 AGENTS.md（工作规则）

```markdown
## ⚠️ 工作规则

### 不需要确认（可自由操作）
- workspace 下的所有文件

### 需要确认
- 系统配置改动
- 发送外部消息
```

**特征**：重要规则、每次都要遵循

### 写入 skills/references/（特定知识）

```markdown
# 小红书爆款标题公式

1. 数字 + 痛点："3个AI工具帮你省10万设计费"
2. 对比 + 悬念："传统设计 vs AI设计，差距太大了"
```

**特征**：特定领域知识、按需触发、不常用但重要

|---|--------|-------------------|
| **memory/research/** | 调研笔记 | 📥 来源（读取提炼） |
| **memory/insights/** | 深度洞察 | 📥 来源（读取提炼） |
| **memory/*.md** | 每日日记 | 📥 来源（读取提炼） |
| **MEMORY.md** | 战略思维 | ✅ 目标（写入） |
| **AGENTS.md** | 工作规则 | ✅ 目标（写入） |

| **USER.md** | 人设 | ❌ 不动 |

---

## 写入检查清单

写入前确认：

- [ ] 这是新知识，不是重复已有内容
- [ ] 不是 Dan 的人设改变（那个要 Dan 确认）
- [ ] 选对了目标文件（用上面的决策树）
- [ ] 战略思维 → MEMORY.md
- [ ] 工作规则 → AGENTS.md
- [ ] 特定知识 → skills/references/
- [ ] 内容精简，不冗余

---

## 避免写入的内容

❌ 临时性的聊天记录
❌ 技术配置细节（归阿技管）
❌ 已经在其他地方记录过的知识
❌ 与 IP 打造无关的内容
❌ Dan 的人设（要 Dan 亲自确认）
